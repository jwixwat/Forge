# v0.1 Closeout and v1.1 Deferred Items

Version: `0.1.0-draft`  
Date: `2026-03-03`

## Purpose

Capture what is considered complete for v0.1 closeout and explicitly mark deferred semantics that remain in v1.1 scope.

## v0.1 Closeout Decisions

1. Canonical event ledger is mandatory input for gate evaluation.
2. Ledger JSONL readback is required for fresh-process replay integrity (events and typed artifacts).
3. Gate catalog is reconciled between runtime implementation and spec matrix (`REFINT`, `REPLAYSTATE` now documented).
4. Governor-related multiplier claims are profile-checked for implemented profiles:
   - `SG_CALIBRATION_GUARD`: ladder-based (`normal/throttle/strong_throttle/freeze`)
   - `SG_SENSOR_UNRELIABLE_GUARD`: fixed `0.10`
   - `SP_*` panic profiles: fixed `0.00`

## Deferred to v1.1

These are intentionally not required to close v0.1:

1. Persistent calibration-window governance authority (escalation windows, stratum persistence counters, freeze decay/clear policy).
2. Runtime enforcement of anchor/shadow pressure escalation mechanics under calibration guard (quota pressure accounting).
3. Profile-specific deterministic confirmation-path requirements beyond multiplier checks (for example rubric confirmation workflow controls in sensor-unreliable guard).
4. Full cross-run operational dashboards and release reports for calibration/governor behavior (contract exists; ops reporting hardening deferred).

## Rationale

v0.1 focuses on integrity spine correctness (canonical events, replay determinism, binding semantics, and auditable mutations).  
v1.1 remains the right activation point for persistent calibration-governor operational behavior and quota-pressure control loops.
