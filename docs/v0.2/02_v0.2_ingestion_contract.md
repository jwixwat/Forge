# v0.2 Ingestion Contract

Status: `draft-implemented`  
Date: `2026-03-04`

## Purpose

This document records the implemented v0.2 ingestion surfaces and validation behavior.
It is scoped to schema/types/validation/registry loading only.

The current starter bundle is no longer a placeholder-only scaffold. It contains a semantically real regex micro-corpus:

- six regex commitments
- per commitment: `M1` direct check, `M2` boundary/minimal-pair check, `T1` teaching/remediation item
- two parallel holdout forms with no dual-use
- one fixed-format closed-book anchor form
- one ultra-short shadow form
- one seeded adversarial minimal-pair generator
- sentinel fixtures for both the slots and MCQ rubrics

## Measurement Contract (Typed and Enforced)

The measurement contract is implemented as first-class typed fields, not metadata:

- `MeasurementSurfaceSpec`
  - `measurement_surface_id`
  - `observation_schema_ref`
  - `response_schema_ref`
  - `rubric_ref`
  - `observation_schema_semantics_hash`
  - `canonicalization_hash`
  - `parse_ir_hash`
  - `rubric_semantics_hash`
  - `obs_binding` (`obs_encoder_version`, `hypothesis_space_hash`)
- `ObservationSchemaSpec`
  - `features`
  - `outcome_surface_feature_ids`
  - `auxiliary_feature_ids`
  - starter regex bundle currently enriches auxiliary observables without changing outcome surfaces:
    - `obs_schema_mcq_v1` adds `selected_choice_id`
    - `obs_schema_slots_v1` adds `slot_error_mask`
- `ResponseSchemaSpec`
  - `authoring_status` (`active_supported` | `reserved`)
  - `payload_schema`
  - `canonicalization_steps`
  - `parse_ir`
- `RubricSpec`
  - `scoring_rules`
  - `observation_emission`
  - `rubric_semantics_version`
- `ProbeFamilySpec.calibration_contract`
  - `inclusion_rules`
  - `strata_axes` (`axis_id`, `derivation_source`, `derivation_path`, `allowed_values`, `unknown_value`)
  - `calibration_target_projection` (`projection_id`, `projection_type`, `source_surface`, `mapping_rules`)
  - `calibration_pool_policy`
    - `baseline_assistance_modes`
    - `monitor_only_assistance_modes`
    - `stratify_by_assistance_mode`
- `ProbeFamilySpec.invariance_contract` (required for `family_kind=measurement`)
  - `axes` (`axis_id`, `expectation`, `rationale`)
  - optional `operational_constraints` (`constraint_id`, `source_path`, `comparator`, `value`, `tolerance`)
- `GeneratorSpec.invariance_contract`
  - `axes` (`axis_id`, `expectation`, `rationale`)
  - optional `operational_constraints` (`constraint_id`, `source_path`, `comparator`, `value`, `tolerance`)
- `ProbeFamilySpec.assistance_contract`
  - `allowed_assistance_modes`
  - `diagnostic_eligible_assistance_modes`
  - `measurement_preserving_assistance_modes`
  - `tool_use_allowed`
- explicit surface refs on delivery contracts
  - `items[].measurement_surface_ref`
  - `generators[].grading_contract.measurement_surface_ref`
- explicit factor-target bindings on authored probes
  - `items[].target_factor_binding`
    - `primary_target_factors`
    - optional `secondary_target_factors`
  - `generators[].target_factor_binding`
    - `primary_target_factors`
    - optional `secondary_target_factors`
- typed grading operands for content
  - `items[].grading_material`
    - `response_kind`
    - `slot_answer_key` for `slots`
    - `correct_choice_id` for `mcq`
    - `numeric_answer` for `numeric`
  - `generators[].grading_contract.solution_material_contract`
    - `response_kind`
    - `derivation_source`
    - `source_path`
    - `required_fields`
- typed form delivery semantics
  - `forms[].delivery_role` (`regular` | `holdout` | `anchor` | `shadow`)
  - `forms[].fixed_form`
  - `forms[].closed_book_only`
  - `forms[].max_presented_items`
- typed adversarial-generator narrowing contract
  - `generators[].adversarial_contract`
    - `generation_mode = seeded_minimal_pairs`
    - `perturbation_axes`
    - `max_perturbation_axes_per_instance`
- bundle-level `feedback_policy_precedence`
  - frozen order: `item -> form -> probe_family -> commitment -> channel_default`

Validation enforces:

- outcome surface non-empty and subset of declared features
- auxiliary features subset of declared features
- outcome/auxiliary disjointness
- measurement family calibration contract presence
- measurement family invariance contract presence
- inclusion channels subset of family allowed channels
- `require_closed_book=true` requires `eligible_assistance_modes == ["closed_book"]`
- unique strata axis ids
- derivation source enum validity
- non-empty strata vocabularies and unknown fallback
- projection source-feature references confined to the declared outcome surface
- invariance axis expectation enum validity (`must_hold` | `may_vary`)
- measurement-family items and generators must bind to exactly one declared measurement surface with matching:
  - `response_schema_ref`
  - `rubric_ref`
  - observation-schema semantics hash
  - canonicalization hash
  - parse-IR hash
  - rubric semantics hash
- rubric semantics are mechanically recomputed from rubric structure and must match the authored `rubric_semantics_version`
- response schemas marked `active_supported` are currently restricted to `slots` and `mcq`; richer response kinds remain reserved at v0.2
- measurement-surface closed-contract checks enforce:
  - scoring-rule source paths compatible with the referenced response schema
  - parse-IR uncertainty fields declared in parse-IR output schema
  - observation-emission features confined to the referenced observation schema
  - required and outcome-surface observation features emitted
- derivation-path syntax and source namespace constraints:
  - `attempt_field -> attempt.*`
  - `telemetry_derived -> attempt.* | telemetry.*`
  - `ir_tags -> item.tags.* | family.tags.* | commitment.tags.*`
  - `feedback_trace -> feedback.*`

## Strictness Model

- Unknown keys on core typed entities are rejected.
- Unknown keys are allowed only inside explicit `metadata` objects.

## Release Hash Canonicalization

`release_hash` is a semantic snapshot hash, not a raw file-byte hash.

- Top-level registry list ordering is normalized before hashing:
  - `commitments`, `factors`, `edges`, `observation_schemas`, `response_schemas`,
    `rubrics`, `measurement_surfaces`, `probe_families`, `items`, `forms`,
    `generators`, `delivery_artifacts`, `feedback_policies`,
    `channel_default_feedback_policies`, `content_migrations`
- Each registry is sorted by its stable ID field before hash computation.
- Known nested set-like arrays are normalized by explicit schema path:
  - `aliases`, `supersedes`, `superseded_by`
  - `allowed_channels`, `measurement_surface_refs`
  - `form_memberships`, `channel_tags`, `role_tags`, `form_tags`
  - `eligible_channels`, `eligible_assistance_modes`, `exclude_residual_flags`
- Truly ordered nested lists remain order-sensitive.

Implication:

- Reordering top-level registries does not create a new release hash.
- Reordering normalized nested set-like lists does not create a new release hash.
- Reordering truly ordered nested lists (for example generator template order) changes the release hash.

## Referential Integrity Hardening

Additional hard rejects enforced in v0.2.1 hardening:

- probe-family item sources must be family-consistent:
  - `static_bank` items must belong to the family
  - `form_bank` form items must belong to the family
  - `generator_bank` generators must belong to the family
- item contracts must align with family contracts:
  - `item.measurement_surface_ref` must be declared by the family
  - `item.response_schema_ref` / `item.rubric_ref` must match the selected surface exactly
  - item/rubric/response-schema combinations must resolve uniquely to one surface
  - item grading material must be present and consistent with the response kind and rubric rule family
- factor `applies_to` references must resolve to known commitment/family IDs
- item and generator target-factor bindings must:
  - resolve to known factor IDs
  - be non-empty on `primary_target_factors`
  - be subsets of the owning family `target_factors`
- commitment lineage (`supersedes`/`superseded_by`) references are validated for:
  - known IDs
  - no self-reference
  - reciprocity
  - cycle absence
- factor lineage (`supersedes`/`superseded_by`) references are validated for:
  - known IDs
  - no self-reference
  - reciprocity
  - cycle absence
- form-item membership is bidirectionally enforced:
  - `forms[].items` must match `items[].form_memberships` reciprocity
- generator grading contracts must match family response-schema and observation-schema contracts through a declared measurement surface
- generator grading contracts must declare typed solution-material derivation for the grading operands required by the referenced rubric
- delivery artifacts (`sentinel_fixture`, `anchor_bank`) must bind only to known rubrics / generators / measurement surfaces
- items may not be both holdout-form and regular-form members unless `dual_use_allowed=true`
- items may not be dual-used across anchor/regular, shadow/regular, or adversarial/regular delivery roles unless `dual_use_allowed=true`
- delivery artifact tags are validated against delivery tag vocab

## Channel Tag Semantics

Channel-defining tags are explicit and singular per item:

- `anchor -> A_anchor`
- `measurement -> B_measurement`
- `learning -> C_learning`
- `shadow -> D_shadow`

Rules:

- exactly one channel-defining tag per item
- item channel must be allowed by its probe family
- family `channel_constraints` must cover every `allowed_channel`
- strict anchor/shadow semantics are enforced by channel row, not by family kind:
  - any `A_anchor` or `D_shadow` constraint row must be `feedback_mode=none`, `hints_allowed=false`, `max_hints=0`
- family kind has channel coherence constraints:
  - `teaching` -> `C_learning` only
  - `anchor` -> `A_anchor` only
  - `shadow` -> `D_shadow` only
  - `adversarial` -> `D_shadow` only
- forms must declare explicit `evidence_channel`
- all form items must map to the form `evidence_channel`
- role tags (`holdout`, `adversarial`) are non-channel modifiers
- forms must also declare explicit delivery semantics:
  - `anchor` forms must be `fixed_form=true`, `closed_book_only=true`, `consumption_policy=none`, `evidence_channel=A_anchor`
  - `shadow` forms must be `fixed_form=true`, `max_presented_items<=3`, `consumption_policy=none`, `evidence_channel=D_shadow`
  - `holdout` forms must be non-learning and consumable (`retire_on_use` or `rotate_forms`)
- items may not be reused across mixed form delivery roles unless `dual_use_allowed=true`
- adversarial probe families must source from `generator_bank`
- generators on adversarial families must declare a narrow adversarial contract with seeded minimal-pair perturbation axes

## Measurement Surface Identity Policy

Registry policy is measurement-surface semantic lock across loaded releases:

- fingerprinting is based on canonicalized measurement semantics:
  - observation-schema semantic hash
  - response canonicalization hash
  - parse-IR hash
  - rubric semantics hash
  - `response_schema_ref`
  - `rubric_ref`
  - obs binding (`obs_encoder_version`, `hypothesis_space_hash`)
- auxiliary-only observation-schema additions remain allowed only when the recomputed measurement-surface fingerprint is unchanged
- any measurement-surface semantic change under the same `measurement_surface_id` is rejected as repoint
- semantically duplicate measurement surfaces under different IDs are rejected; authors must reuse the same `measurement_surface_id` when semantic hashes and obs binding are identical
- optional persisted measurement-surface identity index (`measurement_surface_identity_index_path`) extends this guard across process restarts

## Calibration Pool Policy

- measurement-family calibration contracts must declare a `calibration_pool_policy`
- `baseline_assistance_modes` are the core calibration baseline
- `monitor_only_assistance_modes` may be tracked as monitor-only strata but are not part of the baseline pool
- baseline modes must be:
  - a subset of calibration inclusion modes
  - a subset of family diagnostic-eligible assistance modes
  - a subset of family measurement-preserving assistance modes
- the starter bundle now uses:
  - baseline: `closed_book`
  - monitor-only: `open_book`, `tool_assisted`, `mixed`

## Tag Vocabulary Contract

- `bundle.tag_vocab` is required:
  - `item_tags`, `family_tags`, `commitment_tags` map tag keys -> allowed values
  - `delivery_tags.form_tags`, `delivery_tags.generator_tags`, `delivery_tags.artifact_tags` reserve delivery-level tag vocabularies
- entity tags (`items[].tags`, `probe_families[].tags`, `commitments[].tags`, optional `forms[].tags`, optional `generators[].tags`) are validated against `tag_vocab`
- `ir_tags` derivation paths are restricted to:
  - `item.tags.<key>`
  - `family.tags.<key>`
  - `commitment.tags.<key>`
- `ir_tags` derivation keys must be declared in `tag_vocab`
- metadata paths are not allowed for calibration strata derivation
